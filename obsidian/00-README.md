# Vault Obsidian — Cerebro del Proyecto

Memoria extendida optimizada para consumo por IA. Cada nota sigue principios de densidad de información y token-efficiency.

## Estructura del Vault

| Carpeta | Proposito | Frecuencia de acceso IA |
|---------|-----------|------------------------|
| 01-META | Reglas del vault, indices, mapas de contexto | Cada sesion |
| 02-Agentes | Definiciones de agentes, prompts system, capacidades | Por tarea |
| 03-Estandares | Stack, convenciones, decisiones arquitectonicas | Referencia |
| 04-Contexto | Estado actual, roadmap, metricas del proyecto | Diaria |
| 05-Analisis | ADRs, investigaciones, decisiones tecnicas | Baja |
| 06-Testing | Estrategia, cobertura, planes de prueba | Por feature |
| 07-Deployments | Historial, runbooks, observaciones ops | Por deploy |
| 08-Templates | Plantillas reutilizables para notas nuevas | Creacion |
| 99-Archive | Notas obsoletas, historial de cambios | Nunca (solo humanos) |

## Reglas de uso para IA

1. **Leer en orden jerarquico**: 01-META → 02-Agentes → 03-Estandares → 04-Contexto
2. **No asumir contexto no presente**: Si no esta en el vault, no existe
3. **Actualizar despues de cambios**: Cada decision operativa nueva se registra
4. **Preferir links a repeticion**: Usar [[Nota]] en lugar de copiar contenido
5. **Densidad sobre verbosidad**: Menos tokens, mas informacion util
